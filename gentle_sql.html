Introduction to SQL

Purpose

This is a gentle introduction of Structured Query Language for engineers in Hydraulic Section.

Introduction

The sewer system information is stored in Oracle Spatial Database. We use the Open Spatial Munsys software, which is used on top of AutoCAD, to enter and update spatial data in a graphical environment. We use many tools that use SQL in the background. Hence, I have used examples to gently introduce the capability of SQL.

This demonstration uses SQLPLUS program. But any database program is similar because SQL is an ANSI standard. Few variations exist due to different implementations.

Logging in

  1. Start Windows Powershell

  2. Did you get this?

  3. sqlplus username/password@MUNSYSPDB

  4. SELECT sysdate FROM DUAL;
  5. If you get date as a response, you are connected to the Oracle.

  6. Now you can try many examples.

Examples

Date Check

SELECT sysdate FROM DUAL;

Pipes

SELECT gid, pipe_dia, geom_length FROM mboe.sp_sewgpipe;

This will take long to show 75000 records on your screen.

It is better to count first.

SELECT COUNT(*) FROM mboe.sp_sewgpipe;

Let us use filter to limit to 9 records

SELECT gid, pipe_dia, geom_length FROM mboe.sp_sewgpipe WHERE ROWNUM < 10;

What column names (fields) are there?

DESC mboe.sp_sewgpipe;

The owner field ...

SELECT owner FROM mboe.sp_sewgpipe;

Ooops, 75000 records on my screen is not useful.

Grouping and counting

SELECT owner, COUNT(*) FROM mboe.sp_sewgpipe GROUP BY owner;

Counting number of records is great. But what about length of pipe?

Length

SELECT owner, COUNT(*), SUM(geom_length) FROM mboe.sp_sewgpipe GROUP BY owner;

What is that feet unit for? I like miles.

Miles

SELECT owner, COUNT(*), SUM(geom_length)/5280 FROM mboe.sp_sewgpipe GROUP BY owner;

So many decimals!

Miles 0.00

SELECT owner, COUNT(*), ROUND(SUM(geom_length)/5280,2) FROM mboe.sp_sewgpipe GROUP BY owner;

Only two decimals. Cleaner. But what is the ugly header for?

Alias

SELECT owner AGENCY, COUNT(*) N, ROUND(SUM(geom_length)/5280,2) MILES FROM mboe.sp_sewgpipe GROUP BY owner;

Alias adds clarity. Some of these pipes may have been demolished. Use filter for active pipes.

Active Pipes

SELECT owner AGENCY, COUNT(*) N, ROUND(SUM(geom_length)/5280,2) MILES FROM mboe.sp_sewgpipe WHERE pipe_status='ACT' GROUP BY owner;

Only the active pipes.

Descending Order

SELECT owner AGENCY, COUNT(*) N, ROUND(SUM(geom_length)/5280,2) MILES FROM mboe.sp_sewgpipe WHERE pipe_status='ACT' GROUP BY owner ORDER BY MILES DESC;

Only the active pipes.

Now manholes

SELECT MAX(rim), MIN(rim), MAX(invert), MIN(invert) FROM mboe.sp_sewnode;

Things to try

DESC mboe.sp_sewgpipe;

SELECT DISTINCT(pipe_status) FROM mboe.sp_sewgpipe;

SELECT owner, COUNT(*), SUM(geom_length)/5280 FROM mboe.sp_sewgpipe GROUP BY owner;

SELECT SUM(geom_length)/5280 FROM mboe.sp_sewgpipe WHERE pipe_status='ACT' AND owner='PUC' AND int_year > 1985;

DESC mboe.sp_sewnode;

SELECT DISTINCT(node_type) FROM mboe.sp_sewnode;

SELECT node_type, COUNT(*) FROM mboe.sp_sewnode WHERE node_status='ACT' GROUP BY node_type;

DESC mboe.sp_hh_study;

SELECT COUNT(*) FROM mboe.sp_hh_study;

SELECT st_name_al, ROUND(SUM(geom_length)/5280, 1) FROM mboe.sp_streets GROUP BY st_name_al;

The relational database

SELECT p.pipe_dia, p.geom_length, s.st_name_al, s.seg_cnn FROM mboe.sp_sewgpipe p, mboe.sp_streets s WHERE p.seg_cnn = s.seg_cnn AND ROWNUM < 11;

Conclusion

There is much more you can do with the SQL. Having basic understanding of the RDBM (relational database management) and SQL allows you to use database effectively. It also opens many doors. Many books on SQL are on our floor.
